# 资产负债表公司特定重分类功能需求文档

## 1. 背景与问题

### 1.1 问题描述

在资产负债表重构过程中，不同公司对相同会计科目的使用存在差异，导致相同科目的经济实质不同：

**案例1：其他非流动资产**
- **安井食品**：包含"预付设备款"和"预付投资款"，属于经营资产
- **某些公司**：包含"固定收益类产品"（定期存款、货币性投资产品等），属于金融资产

**案例2：其他流动资产**
- 有些公司：待抵扣进项税（经营资产）
- 有些公司：短期理财产品（金融资产）

### 1.2 影响

如果不区分处理，会导致：
1. **金融资产占比失真**：将金融资产误分类为经营资产，或反之
2. **经营资产分析偏差**：影响对企业真实经营能力的判断
3. **资产结构分析错误**：无法准确评估企业"脱实向虚"程度

### 1.3 解决目标

提供一个灵活的配置机制，允许用户针对特定公司对资产负债表科目进行重分类调整，确保重构结果符合经济实质。

## 2. 功能需求

### 2.1 核心功能

**功能名称**：公司特定科目重分类配置

**功能描述**：
- 用户通过配置文件为特定公司定义科目重分类规则
- 程序在重构资产负债表时自动应用这些规则
- 支持将科目从默认分类移动到目标分类
- 支持部分重分类（按百分比拆分）

### 2.2 配置文件设计

**文件路径**：`config/company_specific_rules.yaml`

**文件结构**：
```yaml
# 公司特定的资产负债表科目重分类规则
company_rules:
  
  # 股票代码作为key
  "股票代码1":
    reclassify:
      # 科目名称
      科目名称1:
        from: 原分类名称
        to: 目标分类名称
        percentage: 重分类比例  # 可选，默认1.0（100%）
        reason: 重分类原因说明  # 可选，用于文档记录
      
      科目名称2:
        from: 原分类名称
        to: 目标分类名称
        
  "股票代码2":
    reclassify:
      科目名称3:
        from: 原分类名称
        to: 目标分类名称
```

**配置示例**：
```yaml
company_rules:
  
  # 示例1：完全重分类
  "603345.SH":
    reclassify:
      其他非流动资产:
        from: 长期经营资产合计
        to: 金融资产合计
        reason: "根据2025年报附注23，主要为固定收益类产品"
  
  # 示例2：部分重分类
  "000333.SZ":
    reclassify:
      其他非流动资产:
        from: 长期经营资产合计
        to: 金融资产合计
        percentage: 0.8  # 80%重分类为金融资产，20%保留为经营资产
        reason: "80%为固定收益产品，20%为预付设备款"
      
      其他流动资产:
        from: 营运资产小计
        to: 金融资产合计
        percentage: 0.6
        reason: "60%为短期理财产品"
  
  # 示例3：多个科目重分类
  "600519.SH":
    reclassify:
      其他非流动资产:
        from: 长期经营资产合计
        to: 金融资产合计
      
      其他应收款:
        from: 营运资产小计
        to: 金融资产合计
        percentage: 0.3
        reason: "30%为应收利息"
```

### 2.3 支持的分类层级

根据资产负债表重构结构，支持以下分类名称：

#### 资产侧分类

**一级分类**：
- `金融资产合计`
- `长期股权投资`
- `经营资产合计`

**二级分类**：
- `周转性经营投入合计`（属于经营资产合计）
- `长期经营资产合计`（属于经营资产合计）

**三级分类**：
- `营运资产小计`（属于周转性经营投入合计）
- `营运负债小计`（属于周转性经营投入合计）

#### 负债及权益侧分类

**一级分类**：
- `有息债务合计`
- `所有者权益合计`

**二级分类**：
- `短期债务`（属于有息债务合计）
- `长期债务`（属于有息债务合计）
- `归属于母公司股东权益合计`（属于所有者权益合计）
- `少数股东权益`（属于所有者权益合计）

**说明**：
- `from` 和 `to` 可以是任意层级的分类
- 程序会自动处理层级关系和金额汇总

### 2.4 重分类规则

#### 规则1：完全重分类（percentage = 1.0 或不指定）

```yaml
其他非流动资产:
  from: 长期经营资产合计
  to: 金融资产合计
```

**处理逻辑**：
1. 从 `长期经营资产合计` 中移除 `其他非流动资产` 的全部金额
2. 将 `其他非流动资产` 加入到 `金融资产合计` 中
3. 重新计算相关的小计和合计项

#### 规则2：部分重分类（0 < percentage < 1.0）

```yaml
其他非流动资产:
  from: 长期经营资产合计
  to: 金融资产合计
  percentage: 0.8
```

**处理逻辑**：
1. 从 `长期经营资产合计` 中减去 `其他非流动资产` 金额的 80%
2. 在 `金融资产合计` 中新增一个计算项：`其他非流动资产(重分类部分)` = 原金额 × 80%
3. `其他非流动资产` 保留在原位置，但金额调整为原金额 × 20%
4. 重新计算相关的小计和合计项

#### 规则3：跨层级重分类

```yaml
# 从三级分类移到一级分类
预付款项:
  from: 营运资产小计
  to: 金融资产合计
```

**处理逻辑**：
1. 从 `营运资产小计` 中移除 `预付款项`
2. 自动向上更新 `周转性经营投入合计` 和 `经营资产合计`
3. 将 `预付款项` 加入 `金融资产合计`
4. 自动向上更新 `资产总额`

### 3.4 错误处理

**场景1：科目不存在**
```
错误：科目"其他非流动资产"在数据中不存在
建议：检查科目名称是否正确，或该公司是否有此科目
```

**场景2：分类名称错误**
```
错误：分类"长期经营资产"不存在
可用分类：金融资产合计、长期股权投资、经营资产合计、周转性经营投入合计、长期经营资产合计...
建议：检查配置文件中的分类名称
```

**场景3：科目不在指定的 from 分类中**
```
错误：科目"其他非流动资产"不在"营运资产小计"中
实际位置：长期经营资产合计
建议：更新配置文件中的 from 字段
```

**场景4：percentage 超出范围**
```
错误：percentage 必须在 0 到 1 之间，当前值：1.5
建议：检查配置文件中的 percentage 字段
```

## 4. 使用场景

### 4.1 场景1：发现科目分类不当

**步骤**：
1. 用户运行 `python main.py 603345.SH --annual-ttm`
2. 查看生成的资产负债表，发现"金融资产占比"异常高
3. 查阅该公司年报附注，发现"其他非流动资产"主要是固定收益产品
4. 在 `config/company_specific_rules.yaml` 中添加重分类规则
5. 重新运行程序，验证结果

### 4.2 场景2：批量处理多个公司

**步骤**：
1. 用户分析多个公司，发现某些行业普遍存在分类问题
2. 在配置文件中为这些公司统一添加规则
3. 批量运行程序，所有公司自动应用各自的规则

### 4.3 场景3：部分重分类

**步骤**：
1. 用户发现某科目既包含金融资产又包含经营资产
2. 根据附注估算比例（如80%金融资产，20%经营资产）
3. 在配置文件中设置 `percentage: 0.8`
4. 运行程序，验证拆分结果
