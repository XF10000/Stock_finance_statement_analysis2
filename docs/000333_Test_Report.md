## 关键发现

### 1. 数据转置是关键步骤

**问题**: Tushare API返回的数据格式是"日期为行，字段名为列"，但重构函数期望的是"字段名为行，日期为列"。

**解决方案**: 在调用重构函数前，必须对数据进行转置处理。

```python
# 转置数据
balance_data_indexed = balance_data.set_index('报告期')
balance_data_transposed = balance_data_indexed.T.reset_index()
balance_data_transposed.columns = ['字段名'] + list(balance_data_transposed.columns[1:])
```

### 2. 参数传递顺序很重要

**问题**: 利润表重构函数的参数顺序容易混淆。

**正确调用**:
```python
restructure_income_statement(income_data, equity_data, equity_cost_rate)
```

**错误示例**:
```python
# ❌ 错误：参数顺序不对
restructure_income_statement(income_data, balance_data, balance_result)
# 这会导致 equity_cost_rate 参数接收到一个 DataFrame 而不是 float
```

### 3. equity_data参数需要重构后的数据

**问题**: 利润表重构函数的 `equity_data` 参数需要的是重构后的资产负债表数据，而不是原始数据或转置后的数据。

**原因**: 需要从重构后的资产负债表中获取"所有者权益合计"项目的数据。

## 建议

### 1. 文档更新

建议在重构函数的文档中明确说明：
- 输入数据的格式要求（字段名为行，日期为列）
- 参数的含义和类型
- `equity_data` 参数需要重构后的资产负债表数据

### 2. 自动化数据准备

建议创建一个辅助函数来自动完成数据转置和格式准备：

```python
def prepare_data_for_restructure(df: pd.DataFrame, date_column='报告期') -> pd.DataFrame:
    """准备数据格式用于重构"""
    df_indexed = df.set_index(date_column)
    df_transposed = df_indexed.T.reset_index()
    df_transposed.columns = ['字段名'] + list(df_transposed.columns[1:])
    return df_transposed
```

### 3. 数据验证

建议在重构函数开始时添加数据格式验证：

```python
def validate_input_format(df: pd.DataFrame) -> bool:
    """验证输入数据格式是否正确"""
    if '字段名' not in df.columns:
        raise ValueError("输入数据必须包含'字段名'列")
    # 检查其他列是否都是日期格式
    for col in df.columns[1:]:
        if not col.replace('年度', '').isdigit():
            raise ValueError(f"列名 {col} 不是有效的日期格式")
    return True
```
