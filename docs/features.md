# 功能清单

## 📊 数据更新功能

### 1. 全量初始化
```bash
python update_financial_data.py --init
```
**功能**：
- 获取全A股列表（约5,100只股票）
- 获取每只股票的完整历史财务数据
- 包括：资产负债表、利润表、现金流量表、财务指标、分红数据
- 自动计算核心指标（年报 + TTM）
- 支持断点续传：`--resume 000333`

**适用场景**：首次使用系统

---

### 2. 增量更新（最新季度）
```bash
python update_financial_data.py --update-latest
```
**功能**：
- 智能判断目标季度（基于当前月份）
- 批量检查缺失数据（0.4秒完成5,191只股票）
- 只对缺失的股票调用API（节省80%+ API调用）
- 自动计算核心指标

**智能季度判断规则**：
| 当前月份 | 目标季度 | 说明 |
|---------|---------|------|
| 2-4月 | 上年Q4 | 例如：2026年3月 → 2025Q4 |
| 5-7月 | 本年Q1 | 例如：2026年6月 → 2026Q1 |
| 8-10月 | 本年Q2 | 例如：2026年9月 → 2026Q2 |
| 11-1月 | 本年Q3 | 例如：2026年12月 → 2026Q3 |

**适用场景**：定期更新（建议每周或每月运行一次）

---

### 3. 单只股票更新
```bash
# 增量更新（最新季度）
python update_financial_data.py --update-stock 000001

# 完整更新（全部历史）
python update_financial_data.py --update-stock 000001 --full
```
**功能**：
- 股票代码自动补全（000001 → 000001.SZ）
- 支持增量和完整两种模式
- 自动计算核心指标

**适用场景**：快速更新单只股票数据

---

### 4. 分红数据更新
```bash
# 更新所有股票的分红数据
python update_financial_data.py --update-dividend

# 更新单只股票的分红数据
python update_financial_data.py --update-stock-dividend 000333
```
**功能**：
- 智能季度判断：根据每只股票财务数据最新季度获取分红
- 只更新缺失的分红数据
- 避免重复获取
- 支持单只股票快速更新（新增）

**示例**：
- 如果某股票财务数据最新到2025Q3 → 获取2025Q3及以前的分红
- 如果某股票财务数据最新到2025Q4 → 获取2025Q4及以前的分红

**适用场景**：
- 批量更新：补充或更新所有股票分红数据
- 单只更新：快速更新某只股票的分红数据

---

### 5. 核心指标重算
```bash
python update_financial_data.py --recalculate-all
```
**功能**：
- 清空现有核心指标
- 批量读取财务数据
- 重新计算年报和TTM指标
- 更新分位数排名

**适用场景**：指标计算逻辑更新后，需要重新计算所有指标

## 🎯 核心指标说明

系统自动计算以下5大核心指标：

### 1. 应收账款周转率对数 (ar_turnover_log)
- **计算公式**：log(营业收入 / 应收账款平均余额)
- **含义**：衡量应收账款周转速度
- **越高越好**：说明回款快，资金周转效率高

### 2. 毛利率 (gross_margin)
- **计算公式**：(营业收入 - 营业成本) / 营业收入
- **含义**：衡量产品盈利能力
- **越高越好**：说明产品附加值高

### 3. 长期经营资产周转率对数 (lta_turnover_log)
- **计算公式**：log(营业收入 / 长期经营资产)
- **含义**：衡量长期资产使用效率
- **越高越好**：说明资产利用率高

### 4. 净营运资本比率 (working_capital_ratio)
- **计算公式**：(流动资产 - 流动负债) / 总资产
- **含义**：衡量短期偿债能力和营运资金充裕度
- **适中为好**：过高说明资金闲置，过低说明流动性风险

### 5. 经营现金流比率 (ocf_ratio)
- **计算公式**：经营活动现金流量净额 / 营业收入
- **含义**：衡量收入转化为现金的能力
- **越高越好**：说明现金回收能力强

**分位数排名**：
- 系统自动计算每个指标在全市场的分位数排名
- 分位数越高，说明该指标在市场中表现越好
- 范围：0-100

## 📝 注意事项

### 1. API限流
- Tushare Pro 限制：150次/分钟
- 系统自动限流，无需手动控制
- 如遇限流错误，系统会自动重试

### 2. 数据完整性
- 新上市股票可能缺少历史数据
- 部分股票可能暂停披露财务数据
- 失败的股票会在日志中记录，下次更新时自动重试

### 3. 季度数据发布时间
| 季度 | 截止日期 | 通常发布时间 |
|------|---------|------------|
| Q1 | 3月31日 | 4-5月 |
| Q2 | 6月30日 | 7-8月 |
| Q3 | 9月30日 | 10月 |
| Q4 | 12月31日 | 次年3-4月 |

**说明**：系统的智能季度判断已考虑这个时间差

### 4. 股票代码格式
系统支持两种格式：
- **简化格式**（推荐）：`000333`、`600519`
- **完整格式**：`000333.SZ`、`600519.SH`

**自动补全规则**：
- `000`、`002`、`003`、`300` 开头 → `.SZ`（深圳）
- `600`、`601`、`603`、`605`、`688` 开头 → `.SH`（上海）
