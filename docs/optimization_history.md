## 1️⃣ 总股本数据优化

**时间**: 2026年3月18日  
**版本**: v1.5

### 问题
- 系统单独调用API获取总股本数据
- 每只股票额外消耗1次API调用
- 全A股约5,100只股票 = 5,100次额外调用

### 解决方案
直接使用资产负债表中的"期末总股本"字段

### 实施内容
1. 修改 `financial_data_manager.py`
   - 从资产负债表读取总股本
   - 移除单独的总股本表

2. 修改 `update_financial_data.py`
   - 移除总股本API调用
   - 只保留财务四表获取

3. 修改 `html_report_generator.py`
   - 从资产负债表获取总股本数据

### 效果
- ✅ 减少5,100+次API调用
- ✅ 简化数据库结构
- ✅ 提升更新速度

## 2️⃣ 分红数据优化

**时间**: 2026年3月18日  
**版本**: v1.6

### 问题
- HTML报告从Excel文件读取分红数据
- 经常出现"未找到分红送股文件"警告
- 数据源不统一

### 解决方案
直接从数据库读取分红数据

### 实施内容
修改 `html_report_generator.py` 的 `_get_dividend_data` 方法：
```python
def _get_dividend_data(self, ts_code: str) -> Optional[pd.DataFrame]:
    """从数据库读取分红数据"""
    return self.db_manager.get_dividend_data(ts_code)
```

### 效果
- ✅ 消除文件缺失警告
- ✅ 数据源统一（全部从数据库）
- ✅ 提升数据一致性

## 3️⃣ 增量更新优化

**时间**: 2026年3月19日  
**版本**: v2.0

### 问题分析

#### 问题1：盲目尝试未发布的季度
```
当前是3月中旬，2026Q1都还没结束
系统却判断2026Q1是最新季度
→ 尝试获取20260331数据
→ 大量API调用返回0条记录
```

#### 问题2：逐只查询效率低
```
检查5,191只股票是否有最新数据
→ 逐只执行SQL查询
→ 耗时较长
```

#### 问题3：无差别API调用
```
所有股票都调用API
→ 包括已有最新数据的股票
→ 浪费API积分
```

### 解决方案

#### 优化1：智能季度判断（财务数据）
根据当前月份判断应该尝试的季度：
```python
if 2 <= month <= 4:
    target_quarter = f"{year-1}1231"  # 2-4月 → 上年Q4
elif 5 <= month <= 7:
    target_quarter = f"{year}0331"    # 5-7月 → 本年Q1
elif 8 <= month <= 10:
    target_quarter = f"{year}0630"    # 8-10月 → 本年Q2
else:  # 11, 12, 1
    target_quarter = f"{year}0930"    # 11-1月 → 本年Q3
```

**示例**：
- 当前：2026年3月 → 目标季度：2025Q4 ✅
- 而不是：2026Q1（还未结束）❌

#### 优化2：批量检查
单次SQL查询检查所有股票：
```python
query = f"""
    SELECT DISTINCT ts_code 
    FROM balancesheet 
    WHERE ts_code IN ('{codes_str}') 
    AND end_date = '{target_quarter}'
"""
```

**性能**：
- 5,191只股票仅需0.4秒
- 相比逐只查询提升1000x+

#### 优化3：按需API调用
只对缺失数据的股票调用API：
```python
stocks_need_update = [s for s in stocks if s['ts_code'] not in existing_stocks]
```

**效果**：
- 典型场景：80%+股票已有数据
- 节省80%+ API调用

#### 优化4：分红数据智能获取
根据每只股票财务数据最新季度获取分红：
```python
# 查询该股票的最新财务季度
latest_date = get_latest_financial_quarter(ts_code)

# 只获取该季度及以前的分红
df = client.pro.dividend(
    ts_code=ts_code,
    end_date=latest_date  # 限制范围
)
```

**示例**：
- 股票A财务数据到2025Q3 → 获取2025Q3及以前的分红
- 股票B财务数据到2025Q4 → 获取2025Q4及以前的分红

### 实施内容

1. **修改 `update_financial_data.py`**
   - 添加 `_determine_target_quarter_smart()` 方法
   - 添加 `_batch_check_missing_stocks()` 方法
   - 修改 `update_latest_quarter()` 使用新逻辑
   - 修改 `fetch_dividend_data()` 智能获取分红

2. **添加股票代码自动补全**
   - 添加 `normalize_stock_code()` 函数
   - 支持不带后缀的股票代码（000001 → 000001.SZ）

3. **添加单只股票更新功能**
   - 新增 `--update-stock` 参数
   - 支持增量和完整两种模式

### 性能对比

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 批量检查（5,191只） | 逐只查询 | 单次SQL | 1000x+ |
| API调用 | 全部调用 | 按需调用 | 节省80%+ |
| 季度判断 | 盲目尝试 | 智能判断 | 避免无效调用 |
| 更新耗时 | 40-50分钟 | 30-40分钟 | 减少20%+ |
